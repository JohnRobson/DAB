Path to Solution

Every data scientist wants to solve a specific problem, is very hard to build a "silver bullet" tool to solve any kind of problem, maybe even "solve" something means that 'we can think what the user is looking for'. Maybe the best approach is to build a HELPER & GUIDE tool:

  • Helper: A "time saver" tool with simple and concise commands to help the user to quickly: get the data, clean, analyse and find a solution approach or at least eliminate non-good approaches.
  • Guide: There several ways to do Dada Analysis, is important to consider the number of elements, is the data is labeled or not and what the user is looking for. To guide the user this DAB can use a methodology of the "classification of the best technique" suggesting the most recommended model to be applied for each situation, , .

Classification of the Best Technique

"Akinator" is a good example, after the user sends the DataSet, DAB will ask some questions like:

  1. Do you want to clean your dataset? (this will handle with Wrong, Missing and Repeated values)
  2. What kind of prediction do you want to do? (Classification, Regression, Auto, etc)

The first question will show the parts of the dataset the seams wrong and the second question will show the Best Results for each predictor with graphics, etc. This part will use Machine Learning to figure out the best calibration for each model.

So, similar do "Akinator", DAB will Guide the user Step by Step to find the Correct and Best data analysis and get the Result that he is looking for. And for advanced data scientists DAB at least will save their time by automatically testing and eliminating several statistical and machine learning models that do not fit for that data set.
